# jeremyknows README Process

This repo is the live public GitHub profile repo for **@jeremyknows**.

## What lives here
- `README.md` — the published GitHub profile copy
- `assets/` — images used by the GitHub profile (header, cards, supporting visuals)
- `docs/` — lightweight maintenance notes for future updates

## Canonical source docs
The strategy and brand system were developed in Watson's workspace and remain the source of truth there:

- Brand strategy guide: `~/.openclaw/agents/main/workspace/docs/knowledge/brand/jeremyknows-brand-identity-2026-03-28.md`
- Visual identity guide: `~/.openclaw/agents/main/workspace/docs/knowledge/brand/jeremyknows-visual-identity-2026-03-28.md`
- Final PDF brand guide: `~/.openclaw/agents/main/workspace/docs/knowledge/brand/jeremyknows-brand-guide-v2.pdf`
- README working draft: `~/.openclaw/agents/main/workspace/data/jeremyknows-github-readme-draft.md`

## Relationship to other repos
- `~/projects/jeremyknows` = GitHub profile / public identity on GitHub
- `~/projects/jeremyknows.com` = standalone personal website

These should stay aligned, but this repo is specifically optimized for the GitHub profile surface.

## Update workflow
1. Draft copy changes in the workspace draft if the edit is substantial.
2. Apply approved changes to `README.md` here.
3. Keep image assets in `assets/` if they are used directly by the public profile.
4. If a change affects brand language or design rules, update the canonical brand docs first.

## Current public framing
The published README is built around:
- **Know enough to be dangerous** as the core line
- A blend of AI/operator/community work
- Project cards and a header banner as the primary visual system
